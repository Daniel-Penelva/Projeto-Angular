Criando componente

  • Primeiramente tem que instalar o CLI do angular com: npm install -g @angular/cli
  • Agora podemos criar projetos mais facilmente, utilizaremos o: ng new "nome do projeto"
  • Para testar o projeto basta digitar o comando: ng serve
  • Podemos alterar os arquivos dentro da pasta de app para refletir mudanças no navegador.
  • Para achar a pasta criada basta apenas usar o o comando: cd "nome da pasta"
  • Para abrir a IDE pelo terminal basta apenas usar o comando: code.
  • Vamos entender melhor o que é criado após um "ng new"
  • node_modules: dependencias do projeto
  • src: onde vamos programar a aplicação
  • src/app: pasta principal para componentes, serviços e etc.
  • Os componentes sao divididos em partes: TS, HTML, CSS e testes.
  • As aplicações globais sao: "index.html", "main.ts", "style.css", "test.ts"
  • Na pasta "assets" é para colocar imagens estáticas
  • Na pasta "environmentes" sao os ambientes que o Angular vai rodar.
  • Na pasta "app" é onde vai programar a aplicação, por exemplo, um evento de click (botão).
  • Na pasta "app" também possui arquivos de configuração, de teste, de html (template) e de estilo (css).
  • IMPORTANTE!!! Para criar um componente podemos utilizar o CLI;
  • o comando é: ng generate component "nome_do_componente"
  • Pode criar uma pasta: ng generate component nome_da pasta/nome_do_componente
  • É feito no terminal os comandos - para abrir o terminal basta ir com o botao direito do mouse e clicar em "Abrir terminal integrado".;
  • Todos os arquivos necessários serão criados no projeto;
  • Repare tb que será criado uma pasta com os componentes criados na API;
  • E vai ter um arquivo atualizado (UPDATE) chamado "app.module.ts";
  • Que corresponde ao arquivo central de configuração da aplicação, onde fica declarado esse componente criado;
  • Ou seja, esse arquivo fica mapeando a aplicação;
  • Para importar o componente basta utilizar o seu "selector" em um HTML de outro componente;
  • Esse selector "app-componente1" pode ser visto no caminho denominado "componente1.component.ts"
  • Esse seletor do componente criado que vai ser usado para importar os elementos;

Interpolação de Dados

  • A interpolação de dados é um recurso que vai nos ensinar a trabalhar com os componentes do angular;
  • Vamos criar nossas variaveis no arquivo .ts, dentro da classe;
  • Ou seja, estas variáveis são propriedades (atributos) da classe
  • E então teremos acesso a estes dados no arquivo .html, o template;
  • A impressão é feita através de: duas chaves e dentro das chaves a propriedade;
  • Os estilos em aplicações de Angular podem ser feitos de duas maneiras:
  • Global: utilizando o arquivo "style.css", que fica em src;
  • Scoped: estilo a nível de componente, criamos quando damos um generate;

Compartilhando dados

  • Em angular podemos compartilhar dados do componente pai com o componente filho;
  • Para isso vamos disponibilizar na chamada do componente o nome do dado que será recebido com a seguinte sintaxe: [dado];
  • E no código .ts do componente filho vamos utilizar o decorator @Input, que tem como papel entregar o dado para o template;
  • Vamos criar um componente: ng generate component components/componente3
  • vamos importar esse componente criado para o componente pai que fica no arquivo "app.component.html";
  • Dentro do arquivo "app.component.ts" que vai ter os dados para poder compartilhar;
  • Por exemplo, vou criar uma propriedade chamada "userName" para compartilhar os dados do Joaquim para outros componentes;
  • Porém é preciso importar o modulo, no caso, o "Input" que vai ser declarado dentro do "import" do arquivo "parent-data.components.ts";
  • Depois, vamos chamar e manipular esse Input para compartilhar a propriedade name;
  • Por fim, podemos atribuir um valor a essa propriedade dentro do arquivo "componente3.component.html"
  • Portanto, vão ser utilizados os seguintes arquivos nas respectiva ordem:
  • (1) app.component.ts (2) componente3.components.ts (3) componente3.component.html (4) app.component.html

Diretivas

  • Podem realizar diversas funções no sistema, como aplicar estilos a um elemento no nosso template;
  • Elas começam sempre com ng, o nome fica como: ngAlgumaCoisa;
  • Vamos inserir diretivas no CSS no arquivo "componente4.components.ts" e concatenar as propriedades na tag html;
  • Vamos inserir diretivas de class tb para depois manipular no seu CSS.

Renderização condicional

  • É possivel exibir determinado conteúdo por meio de uma condicional;
  • Utilizamos a diretiva nglf para isso;
  • Os valores podem ser dinâmicos (propriedades), mas podemos realizar outros tipos de comparação;
  • Há a possibilidade também de imprimir um cenário para validação de falso, com o else.

Eventos

  • Podemos ativar eventos nos componentes para disparar algun método;
  • Um evento clássico que utilizamos muito é o click;
  • A sintaxe é: (click)= "algumaFunção()"
  • Os métodos ficam na classe;

Emitindo Eventos

  • Podemos comunicar eventos de um componente filho para o pai;
  • Para isso vamos precisar importar o @Output, que vai fazer a saída do evento do componente filho;
  • Vai precisar também importar o EventEmiter para poder enviar o evento.
  • Na tag de invocação do componente do template, escolhemos um método para executar quando o evento for emitido;

Renderização de Listas

  • Outro recurso importante é o loop em listas;
  • Para isso vamos precisar de uma propriedade com os itens da lista;
  • E no template do componente vamos utilizar o *ngFor.
  • Podemos renderizar template baseado em dados;

Importância das Interfaces

  • Toda entidade que devemos trabalhar precisa de interface;
  • Isso vai tornar nosso código mais simples ao longo do programa;
  • Padronizando é facilmente fazer a manutenção;
  • Vamos implementar uma interface na nossa lista de animais;
  • E fazer um evento que também utilizará este recurso.
  • IMPORTANTE! Vai ser criado um arquivo "Animal.ts" dentro da pasta app.
  • A extensão ".ts" é de typeScript - é sempre baseada em TypeScript.
  • O arquivo "Animal.ts" é uma classe.

Pipe Operations

  • O Pipe Operations são recursos para trabalhar com Strings no template;
  • Podemos realizar diversas funções, como deixar o texto em caixa alta;
  • A sintaxe é: "dado|algumPipeOperator" fica dentro de duas chaves;
  • Dessa maneira nos poupamos de regras de CSS ou manipulação de Strings com JS.

Two way data binding

  • O Two way data binding é um recurso muito interessante para o trabalho com formularios;
  • Basicamente conseguimos alterar props e o template com o preenchimento de inputs;
  • É necessário importar o "FormsModule" no componente principal;
  • Ou seja, será importado no arquivo "app.module.ts" o "FormsModule"
  • Ainda no arquivo "app.module.ts" terá que importar também o "FormsModule" no modulo extra do "imports:[]"
  • E também declarar o ngModel no input, além de preencher também o atributo nome, todos com o mesmo valor.

Services

  • O Service é outra parte fundamental da arquitetura do Angular;
  • Geralmente ficam as requisições para as APIs que utilizamos no projeto;
  • Precisamos criar o service com: ng generate service "nome";
  • Vamos criar uma pasta chamada "services" atraves do comando:
  • ng generate service services/componente12;
  • Vão ser criado dois arquivos: "componente12.service.spec.ts" e "componente12.service.ts";
  • importar no componente e iniciar no constructor;
  • Depois é possivel acessar os métodos dele.

Angular Route

  • Nós vamos criar um novo arquivo para declarar nossas rotas;
  • Vamos criar um arquivo dentro da pasta "app" chamado de "app-routing.module.ts"
  • Nele vamos importar os módulos RouterModule e Routes;
  • Após a definição estas rotas precisam ser importadas em "app.module.ts";
  • E por fim no template principal criamos as rotas;
  • E substituímos os componentes por: "router-outlet"
